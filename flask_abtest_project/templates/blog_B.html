Front-End

React.js, Vue.js, Next.js

제 개인적인 경험이지만, 리액트를 공부하고 있는 도중에 Vue.js에도 흥미를 느끼게 되었습니다. Vue.js는 다른 접근 방식과 유연성을 제공하는 것 같아서 제가 새로운 기술을 배우는 것에 흥미를 느끼게 되었습니다. 그래서 이제는 Vue.js도 공부할 예정입니다. 한편으로는 Next.js에도 관심을 가지게 되었습니다. Next.js는 리액트 기반의 프레임워크로, 서버 사이드 렌더링과 정적 사이트 생성을 지원하여 웹 애플리케이션의 성능과 검색 엔진 최적화를 향상시킬 수 있습니다. 예전에 살짝 다뤄본 경험이 있지만, 이번에는 더 깊이있게 학습하여 제대로 활용해보고 싶습니다.

Back-End

Flask, FastAPI, Django

요즘에는 백엔드 개발과 데이터 분석에도 흥미를 느끼고 있습니다. 특히 Python 언어의 다양한 라이브러리와 프레임워크를 활용하여 이러한 영역들을 탐구하고 있습니다. Flask, FastAPI, Django 등 다양한 백엔드 프레임워크를 공부하고 싶은데, 각각의 특징과 장단점을 이해하고 싶습니다. Flask는 가벼운 마이크로 프레임워크로, 작은 규모의 애플리케이션을 빠르게 개발할 수 있도록 도와줍니다. FastAPI는 높은 성능과 현대적인 API 개발을 위한 프레임워크로, 비동기 처리와 자동 문서화 기능을 강조합니다. Django는 전체 웹 애플리케이션을 개발하기 위한 프레임워크로, 강력한 기능과 보안성을 제공합니다.

풀스택 개발자란 무엇일까?

풀스택 개발자는 소프트웨어 개발 프로세스에서 프론트엔드와 백엔드 모두를 다루는 개발자를 가리킵니다. 즉, 웹 애플리케이션 또는 웹 사이트의 전체적인 구축과 유지보수를 담당하는 역할

프론트엔드란?

프론트엔드 개발자는 사용자가 직접 상호작용하는 웹 애플리케이션의 인터페이스를 개발합니다. 이는 사용자 경험을 중시하고, 디자인 및 사용자 인터랙션을 구현함으로써 사용자에게 직관적이고 편리한 환경을 제공합니다. 주로 HTML, CSS, JavaScript를 사용하여 웹 페이지를 구축하며, 이를 통해 사용자에게 정보를 시각적으로 전달하고 상호작용을 가능하게 합니다. 최근에는 React, Vue.js, Angular와 같은 프레임워크 및 라이브러리가 많이 사용되고 있어서, 이를 활용하여 웹 애플리케이션을 보다 동적이고 효율적으로 구현할 수 있습니다.

백엔드란?

백엔드 개발자는 서버 측에서 웹 애플리케이션의 기능을 구현하고 데이터를 관리하는 역할을 담당합니다. 주로 서버 사이드 프로그래밍 언어를 사용하여 로직을 작성하며, 데이터베이스와의 상호작용을 통해 사용자의 요청에 응답합니다. 백엔드 개발에서는 데이터베이스 설계와 관리, 사용자 인증 및 보안, API 개발 등 다양한 영역을 다룹니다. 최근에는 Flask, Django, Express.js, Spring 등의 프레임워크와 라이브러리가 사용되며, 이를 활용하여 백엔드 서비스를 구축하고 관리합니다.

서버 및 인프라 관리?

서버 및 인프라 관리는 웹 애플리케이션의 배포와 유지보수를 담당합니다. 클라우드 서비스를 활용하여 서버를 프로비저닝하고, 애플리케이션을 배포하며, 서버의 확장성과 안정성을 유지하는 역할을 수행합니다. 이는 인프라스트럭처의 설정과 관리뿐만 아니라 모니터링, 로그 분석, 보안 패치 등 다양한 작업을 포함합니다. 풀스택 개발자는 서버와 인프라에 대한 기본적인 이해를 갖추고, 이를 통해 애플리케이션의 전체적인 성능과 안정성을 유지하는 데 기여합니다.

왜 풀스택의 길을 걸으려고 하는가?

저는 원래 프론트엔드 개발자 지망생으로써 프론트엔드만 공부하고 있었습니다. 그런데 프로젝트 경험을 하면서 뭔가 한 쪽 면만 맡아서 진행하다보니까 전체적인 그림이 안보여서 좀 답답하다는 느낌이 들어 풀스택으로 방향을 변경했습니다. 근데 백엔드는 Java/spring이 국룰아니냐고 많이들 물어보십니다. 맞습니다. 하지만 저는 데이터분석쪽 대학원까지 욕심이 생겨서 그냥 백엔드 언어를 python으로 고정시키고 python기반의 여러 프레임워크들을 경험해보려고 합니다.
